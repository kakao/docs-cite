Design

Create

participant User participant Cite participant kubernetes participant elasticsearch participant Github User->Cite: create app Cite->kubernetes: upsert kubernetes namespace Cite->elasticsearch: upsert kibana index Cite->Github: upsert github webhook Cite->Github: add cite user as collaborator Cite->kubernetes: upsert kubernetes service
  1. create new service on cite webUI
  2. create/update kubernetes namespace
  3. create/update kibana index on elasticsearch for Pod log
  4. register cite webhook on github repository
  5. add cite account as a collaborator on github repository for source checkout, commit status update, deployment create, deployment status update

Build

participant User participant Cite participant GitHub participant Buildbot participant DockerReg User->GitHub: git push GitHub->Cite: event: push Cite->Buildbot: start build Buildbot->GitHub: commit status: pending GitHub->Cite: event: status(pending) Cite->User: notify: build start Buildbot->Buildbot: build docker image Buildbot->DockerReg: push docker image Buildbot->GitHub: commit status: success GitHub->Cite: event: status(success) Cite->User: notify: build finish
  1. user pushes source to github
  2. github send push event to cite
  3. cite trigger buildbot to build docker image
    • X-GitHub-Event: cite
  4. buildbot start docker build
  5. buildbot update commit status to pending
  6. github send status/pending event to cite
  7. cite send notify : build started
  8. buildbot execute docker image build
  9. buildbot execute docker image push
  10. buildbot update commit status to success or failure
  11. github send status/success or status/failure event to cite
  12. cite send notify : build finished

Deployment

participant User participant Cite participant GitHub participant Kubernetes GitHub->Cite: event: status(success) Cite->GitHub: deployment Cite->GitHub: deployment status: pending GitHub->Cite: event: deployment(pending) Cite->User: notify: deploy start Cite->Kubernetes: create replication controller Cite->Cite: wait for Pods become ready Cite->Kubernetes: update service selector Cite->GitHub: deployment status: success GitHub->Cite: event: deployment(success) Cite->User: notify: deploy success
  1. github send status/success event to cite
  2. cite send create deployment to github
  3. cite send create deployment status/pending to github
  4. github send back deployment status to cite
  5. cite send notify : deploy started
  6. cite create kubernetes replication controller
  7. cite polls for all pods to become ready status
  8. cite update kubernetes service selector
  9. cite send create deployment status/success to github
  10. github send back deployment status to cite
  11. cite send notify : deploy finished
